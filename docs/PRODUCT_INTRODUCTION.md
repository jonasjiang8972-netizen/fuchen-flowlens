# 拂尘 FlowLens 产品介绍

> 文档版本：V1.0  
> 更新日期：2026-07-08  
> 产品定位：云原生 API 旁路安全监控与全生命周期治理平台

## 1. 产品概述

拂尘 FlowLens 是面向企业 API 安全、数据治理与风险运营场景的旁路监测平台。产品通过 Agent、网关日志和事件接入等方式采集真实 API 流量，在不改造业务系统、不串联阻断生产链路的前提下，持续发现 API 资产、识别影子接口和僵尸接口、检测越权与撞库等业务逻辑风险，并把告警、盲区、契约差异转化为可跟踪的治理工单。

FlowLens 的核心理念是“以真实流量为证据”。相比只依赖静态文档、CMDB 或网关配置，FlowLens 从线上请求、响应、身份、调用链路和数据字段中还原 API 的真实运行状态，帮助安全、架构、平台、研发团队共同回答：

- 现在到底有哪些 API 在被访问？
- 哪些 API 返回敏感数据、暴露在公网或缺少责任人？
- 哪些调用方正在越权访问对象、管理端点或高敏接口？
- 哪些接口与 OpenAPI、Proto、CMDB 或设计流程不一致？
- 哪些风险已经处置，哪些仍然超过 SLA 或反复出现？

## 2. 目标用户

FlowLens 面向以下团队和角色：

| 角色 | 关注点 | FlowLens 提供的价值 |
|------|--------|---------------------|
| 安全运营团队 | 高危告警、攻击溯源、处置闭环 | 发现 BOLA、BFLA、撞库、敏感数据暴露等风险，并沉淀审计链 |
| API 平台团队 | API 资产、Owner、生命周期 | 发现影子 API、僵尸 API、未认领接口、契约漂移 |
| 架构治理团队 | 调用链路、服务依赖、设计偏差 | 用真实流量校验业务流程和接口设计一致性 |
| 数据安全团队 | 敏感字段、数据流向、脱敏缺陷 | 识别高敏字段外发、未脱敏响应、导出接口风险 |
| 研发负责人 | 风险修复、SLA、复发检测 | 将风险转化为工单，绑定责任团队并跟踪闭环 |
| 管理层 | 治理成熟度、趋势、业务线排行 | 汇总资产、风险、覆盖率、SLA、Owner 覆盖等治理指标 |

## 3. 产品能力总览

### 3.1 治理驾驶舱

治理驾驶舱面向管理层和负责人，聚合 API 总资产、高危风险、数据暴露面、供应链风险、Owner 覆盖、采集覆盖、身份解析置信度和处置 SLA 等指标。当前前端已提供业务线风险排行、治理趋势、治理成熟度和管理层关注问题入口。

### 3.2 安全工作台

安全工作台聚合资产、采集器和告警视图，提供高危告警、待认领资产、在线采集器、告警趋势、风险分布、高风险资产等关键运营视图。适合安全值班和每日风险巡检。

### 3.3 API 资产中心

资产中心基于真实流量识别 API，包括 REST、GraphQL、gRPC、WebSocket 等协议形态。当前能力包括：

- API 路径归一化，例如将 `/api/v1/order/12345` 归一为 `/api/v1/order/{id}`。
- API 基础属性展示，包括方法、协议、主机、调用量、敏感度、分组、Owner、状态。
- 影子 API、僵尸 API、高敏 API、未认领 API 的标记。
- 资产详情页展示请求统计、调用趋势、敏感字段、关联告警和变更历史。

### 3.4 告警中心

告警中心围绕 API 业务逻辑风险和数据风险展开，当前覆盖：

- BOLA 对象级越权/对象遍历。
- BFLA 功能级越权/管理端点越权。
- 认证失败、撞库、批量账号尝试。
- 异常资源消耗。
- 影子 API、僵尸 API。
- 敏感数据未脱敏、明文传输等数据风险。

告警详情页支持查看风险评分、置信度、来源规则、来源 IP、账号、攻击路径、事件时间线和关联告警。

### 3.5 调用链路

调用链路页面从入口 API、网关、BFF、内部服务和敏感数据节点出发，还原业务流程中的服务依赖和数据流向。当前页面提供：

- 内外网调用拓扑。
- 业务流程和一致性偏差列表。
- 设计偏差标注，如文档缺失、trace 缺口、绕过校验、敏感字段流转。
- 相关 API 资产下钻。

### 3.6 契约一致性

契约一致性模块用真实流量校验 OpenAPI、Proto、CMDB 和业务流程设计，识别：

- 文档缺失接口。
- Schema 漂移。
- 响应字段超范围返回。
- 状态码不一致。
- 非预期调用方。
- 高频旧版本调用。

该模块帮助 API 平台团队发现“线上真实 API”和“设计文档 API”之间的差异。

### 3.7 身份与调用方

身份中心将调用方分为外部用户、内部用户、供应链伙伴、内部服务等主体类型，并通过多种证据源解析身份：

- JWT Subject。
- Partner API Key。
- mTLS SAN。
- Gateway Consumer。
- Cookie、Session、IP、UA 的弱关联。
- 内部用户、Partner、ServiceAccount、网络区域注册表。

平台对弱关联结果标注置信度，避免将不确定主体误判为确定身份。

### 3.8 数据治理

数据治理模块面向敏感字段、数据流转和脱敏缺陷。当前页面展示：

- 敏感字段总览。
- 数据流动图谱。
- 脱敏缺陷看板。
- 合规报告中心预留。

当前检测逻辑已能从查询参数、请求头和响应体标记中识别手机号、身份证、银行卡、Token、密码、地址、邮箱等敏感字段线索。

### 3.9 覆盖率与盲区

覆盖率盲区模块用于明确哪些 API、身份、数据、链路和基础设施还不可见。当前页面覆盖：

- 采集覆盖率。
- Trace 覆盖率。
- 响应体采样覆盖。
- Partner 归属覆盖。
- 采集盲区、身份盲区、数据盲区、链路盲区、责任盲区、证据盲区。
- 盲区清单与修复建议。

### 3.10 采集与链路

采集器管理模块展示 Agent 在线状态、采集模式、集群、QPS、CPU、内存、丢包率、版本和心跳。Agent 侧当前具备：

- 自动环境探测。
- eBPF、Gateway Log 等采集模式入口。
- APIEvent 归一化。
- 批量事件上报。
- 注册和心跳。
- 健康指标采集。

### 3.11 处置闭环

处置闭环将告警、盲区和契约差异转化为治理工单。当前页面包括：

- 开放工单、SLA 风险、Owner 覆盖、复发检测指标。
- 工单列表，包含来源、Owner、等级、状态、SLA、期限和治理动作。
- 闭环审计链，记录告警触发、工单创建、临时处置、修复验证。

### 3.12 检测策略

检测策略模块提供规则可见、可知、可控、可优化能力。当前内置规则包括：

- BOLA 对象遍历检测。
- 认证失效检测。
- BFLA 越权操作检测。
- 敏感数据检测。
- 爬虫行为检测。
- 资源消耗异常检测。
- SSRF 检测。

规则支持启停、查看详情、调整参数和命中次数统计。

### 3.13 AI 应用治理

AI 应用治理当前为 Phase 2 / Phase 3 预留能力，规划覆盖模型 API、Agent 工具调用、RAG 数据访问、Prompt/Context 风险、审计与回放。

## 5. 差异化价值

1. 旁路优先：不改变业务链路，不引入串联阻断风险，适合生产环境先观测后治理。
2. 真实流量证据：以线上请求、响应、身份和链路作为判断依据，而不是只依赖静态资产台账。
3. API 生命周期闭环：从发现、归类、认领、检测、告警、工单到复发观察，形成治理闭环。
4. 业务逻辑风险聚焦：优先覆盖传统 WAF 难以识别的 BOLA、BFLA、撞库、慢速拖库等风险。
5. 面向多角色协同：同时服务安全、平台、架构、数据、研发和管理层。

## 6. 典型使用场景

### 场景一：影子 API 清理

平台通过真实流量发现未登记接口，标记为影子 API，并结合敏感字段、外部入口、Owner 状态计算风险。安全或平台团队可将其转为工单，要求补齐 CMDB、OpenAPI 文档和网关策略。

### 场景二：对象级越权检测

同一账号在短时间内访问大量不同订单 ID，BOLA 引擎识别对象遍历行为，生成高危告警。告警详情展示攻击路径、来源 IP、账号、风险评分和处置建议。

### 场景三：敏感数据外发治理

平台从响应体和字段名中识别身份证、手机号、银行卡、地址等高敏字段，并关联导出接口、调用方和业务流程，帮助数据安全团队定位未脱敏接口和超范围返回。

### 场景四：契约漂移治理

真实流量中出现 OpenAPI 未声明的状态码、响应字段或调用方，契约一致性模块生成差异项，推动研发团队修正文档、客户端适配和接口权限。

### 场景五：采集盲区补齐

覆盖率模块展示某些集群缺 Agent、某些服务缺 trace、某些 Partner 无注册表映射。平台团队据此补齐采集链路，提升告警证据完整度。

## 7. 产品路线建议

### POC 阶段

- 稳定现有前端控制台。
- 打通 Agent 到平台 ingest 的端到端演示。
- 固化 APIEvent 样例和回放脚本。
- 将内存数据逐步迁移到 Postgres/ClickHouse。
- 补齐基础部署和演示文档。

### MVP 阶段

- 完成真实 Gateway Log 和 eBPF 采集可用性验证。
- 完成资产增量发现、Owner 认领、规则命中、告警生成闭环。
- 引入可配置的身份解析器和敏感字段分类器。
- 完成工单闭环和审计记录。
- 建立覆盖率与盲区的真实计算逻辑。

### V1.0 阶段

- 引入更完整的 OWASP API Top 10 检测能力。
- 接入企业 IAM、CMDB、OpenAPI 仓库、网关、SOAR、工单系统。
- 支持多租户、RBAC、审计、报表和合规导出。
- 建立 AI 应用治理能力，包括模型 API、Agent Tool Call 和 RAG 数据访问审计。
