## 1. 一句话介绍

拂尘 FlowLens 是一款云原生 API 旁路安全监控与全生命周期治理平台，帮助企业在不改造业务系统的前提下，看清 API 资产、识别业务逻辑风险、治理敏感数据暴露，并推动风险闭环。

## 2. 标语候选

- 清风拂尘，流量为镜。
- 让每一个 API 风险都有证据、有 Owner、有闭环。
- 不入侵业务链路，看清真实 API 风险。
- 从真实流量出发，重建 API 安全治理视野。
- API 看得见，风险管得住，治理闭得上。

## 3. 核心卖点

### 3.1 旁路部署，低风险接入

FlowLens 优先采用旁路观测方式接入生产环境，避免传统串联安全设备带来的单点故障、性能损耗和业务改造压力。企业可以先建立可见性，再逐步推进治理策略。

### 3.2 真实流量驱动，不只看文档

FlowLens 以线上真实请求和响应为证据，自动发现影子 API、僵尸 API、未认领 API、Schema 漂移和非预期调用方，解决“文档里没有、网关里不全、CMDB 里过期”的 API 资产盲区。

### 3.3 聚焦业务逻辑风险

平台内置 BOLA、BFLA、认证失败、撞库、敏感数据导出等检测能力，面向传统 WAF 难以覆盖的业务逻辑攻击和慢速风险。

### 3.4 数据风险与接口治理一体化

FlowLens 不只看攻击，也看数据。平台可识别手机号、身份证、银行卡、Token、地址、邮箱等敏感字段线索，并关联接口、调用方、业务流程和工单闭环。

### 3.5 从告警到闭环

告警、盲区、契约差异都可以沉淀为治理工单，绑定 Owner、SLA、修复动作和复发观察，避免风险只停留在看板上。

## 4. 客户痛点文案

### 痛点一：API 家底不清

研发迭代快、微服务数量多、网关配置复杂，安全团队很难知道线上到底有哪些 API 在被访问。影子接口、临时接口、历史导出接口经常脱离管理。

FlowLens 通过真实流量持续发现 API 资产，让企业看清“正在被访问的 API”，而不是只看到“文档里声明的 API”。

### 痛点二：传统 WAF 看不懂业务逻辑

对象遍历、越权访问、慢速拖库、撞库攻击往往使用正常协议和合法账号，看起来不像传统攻击特征。

FlowLens 从账号、对象 ID、接口路径、访问频率、身份角色和历史行为中识别业务逻辑异常。

### 痛点三：敏感数据流向不可控

手机号、身份证、银行卡、地址等字段可能通过导出接口、Partner API、后台接口或旧系统被超范围返回。

FlowLens 将敏感字段和 API 资产、调用链路、调用方、工单关联起来，让数据风险可以定位、分派和复核。

### 痛点四：风险无法闭环

很多安全平台能产生告警，但告警到研发修复之间缺少 Owner、证据、SLA 和复发检测。

FlowLens 把告警、盲区、契约差异转为治理工单，让 API 风险进入可管理的闭环流程。

## 6. 典型场景话术

### 6.1 面向 CISO / 安全负责人

FlowLens 帮助您把 API 安全从“告警响应”提升到“治理运营”。它能告诉您哪些 API 高危、哪些数据外发、哪些业务线风险复发、哪些告警超过 SLA，并形成可审计的处置闭环。

### 6.2 面向 API 平台负责人

FlowLens 用真实流量持续校验 API 台账，发现影子 API、僵尸 API、未认领接口和契约漂移，帮助平台团队维护可信 API 资产目录。

### 6.3 面向架构负责人

FlowLens 将入口 API、服务依赖、业务流程和数据流向放到同一张图里，帮助架构团队发现 trace 缺口、设计偏差、非预期调用和服务边界风险。

### 6.4 面向数据安全负责人

FlowLens 能从接口响应和字段特征中识别敏感数据暴露，把敏感字段与调用方、接口、业务流程、导出行为关联起来，支撑数据安全治理和合规审计。

## 10. 客户演示脚本

1. 打开治理驾驶舱，展示 API 总资产、高危风险、数据暴露、供应链风险。
2. 进入 API 资产中心，筛选高敏 API、影子 API 和未认领 API。
3. 打开告警中心，查看 BOLA 或撞库告警详情。
4. 查看调用链路，解释入口 API 到内部服务和敏感数据节点的关系。
5. 打开契约一致性，展示真实流量和 OpenAPI/CMDB 的差异。
6. 打开身份与调用方，展示外部用户、Partner、内部服务的识别证据和置信度。
7. 打开覆盖率盲区，说明哪些证据缺口会影响判断。
8. 打开处置闭环，展示告警如何变成工单、绑定 Owner 和 SLA。
9. 最后回到治理驾驶舱，说明管理层如何持续跟踪治理成熟度。

## 11. 竞争对比表达

FlowLens 不替代网关、WAF、CMDB 或工单系统，而是把它们连接起来：

| 类型 | 常见能力 | FlowLens 补足点 |
|------|----------|-----------------|
| API 网关 | 路由、鉴权、限流 | 基于真实流量发现治理问题和业务逻辑风险 |
| WAF | 特征攻击防护 | 识别合法账号下的越权、遍历、撞库、拖库 |
| CMDB | 静态资产台账 | 用真实调用校验资产是否存在、是否过期、是否缺 Owner |
| APM / Trace | 性能和调用链 | 叠加身份、敏感数据、契约偏差和安全风险 |
| SOAR / 工单 | 编排和处置 | 提供 API 风险证据、Owner、SLA 和复发观察 |

## 12. 注意事项

当前项目处于 POC 到 MVP 演进阶段。对外宣传时建议采用以下表达：

- 可以说：已具备控制台、事件接入、基础检测、资产与告警演示闭环。
- 可以说：支持 Gateway Log/eBPF 等采集模式入口，具备 Agent 上报框架。
- 可以说：部署架构规划包含 Postgres、ClickHouse、Elasticsearch、Kafka、Redis。
- 谨慎说：大规模生产压测、完整 eBPF 生产采集、AI 行为基线、企业系统深度集成仍属于后续增强。
