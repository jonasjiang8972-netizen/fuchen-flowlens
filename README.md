# 拂尘 FlowLens

> 新一代云原生 API 旁路安全监控与全生命周期治理平台

## 项目简介

「拂尘 FlowLens」将中国传统文化中的内敛、纯净，与现代云原生流量技术的洞察、精准完美结合。

- **拂尘（The Whisk）**：纯旁路（Bypass）部署，如清风拂面。在零业务侵入、零性能损耗、零改造风险（三零原则）下，悄悄拂去 API 中的恶意威胁与敏感数据泄露隐患，保障生产环境纯净。

- **FlowLens（流量棱镜）**：Flow 代表流不息的 API 流量，Lens 代表高精度透视。像多维棱镜一样，将混杂、加密的数字化流量深层拆解，隐蔽威胁无所遁形。

## 行业背景与核心痛点

传统串联安全设备（如传统 WAF/网关）在 API 时代面临三大瓶颈：

| 痛点 | 描述 |
|------|------|
| 投鼠忌器 | 串联部署存在单点故障风险，且高并发下深层解析会带来毫秒级延迟，业务无法接受 |
| 资产盲区 | 研发迭代快，"影子 API"、"僵尸接口"泛滥，安全团队对家底模糊不清 |
| 逻辑漏洞 | 越权访问（BOLA/BFLA）、慢速拖库等业务逻辑攻击没有典型特征码，传统 WAF 无法拦截 |

## 技术架构

平台采用"高内聚、低耦合"的架构，确保采集足够轻、分析足够深：

```
┌─────────────────────────────────────────────────────────────┐
│                    多维分析引擎 (The Lens Engine)              │
│         OWASP API Top 10 规则引擎 + AI 行为基线引擎            │
├─────────────────────────────────────────────────────────────┤
│                   高性能数据管道 (The Conduit)                 │
│              Flink + Kafka 流式重组 · TLS 旁路解密             │
├─────────────────────────────────────────────────────────────┤
│                   极致轻量采集层 (The Brushes)                 │
│         SPAN 镜像 · 网关轻量插件 · eBPF 内核级流量捕获          │
└─────────────────────────────────────────────────────────────┘
```

## 核心功能模块

### 1. API 资产全量梳理
- 自动发现未知接口
- 绘制"API 资产活地图"
- 提炼参数结构

### 2. 敏感数据合规治理
- 自动识别响应体中的敏感信息（姓名、手机号、各类凭证）
- 追踪数据流向
- 满足《数据安全法》合规审计

### 3. 业务逻辑风险监测
- 关联上下文，精准识别假冒合法身份的水平/垂直越权
- 恶意爬虫及批量拖库检测

### 4. 脆弱性评估与加固
- 提前感知未授权访问、参数配置不当等隐患
- 指导研发漏洞修复

## 项目演进路线图

| 阶段 | 时间 | 目标 |
|------|------|------|
| 织网观流 | 1-3个月 | 聚焦流量无感采集，盘点全公司 API 资产，消灭"影子接口" |
| 净土拂尘 | 4-6个月 | 上线规则与敏感数据引擎，攻克高危漏洞，与安全事件响应中心（SOC）联动 |
| 全智多维 | 7个月及以后 | 引入 AI 行为基线检测高级威胁，打通 DevSecOps 研发流水线，实现安全左移 |

## 技术栈（规划中）

- **采集层**: eBPF (Cilium/libbpF), SPAN/TAP, Service Mesh Sidecar
- **数据管道**: Apache Kafka + Apache Flink
- **存储**: ClickHouse (流量日志), Elasticsearch (威胁检索), PostgreSQL (元数据)
- **分析引擎**: Python (规则引擎 + ML), Rust (高性能匹配)
- **前端**: React + TypeScript, 可视化 (D3.js/ECharts)
